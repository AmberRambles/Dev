CSS CheatSheet

The principle rule of CSS is that more specific rules will override more general rules.

This means that the CSS rule you make may not make it to the final viewer, if it gets overridden. One workaround is to add !important to the end of a CSS property declaration. Another is to modify your rule to use a more specific selector (or set of selectors).

Another common issue (that I run into) is that certain HTML elements will inherit properties from their container or may pull default properties from a user-agent (web-browser) stylesheet. This may result in an HTML element that does not look as you may expect. (margins magically appearing where they should not) You can use browser dev tools to confirm the issue before implimenting a fix.

Selectors

In CSS, we can access elements using selectors. Selectors allow us to set rules for many elements at once. The order and specificity of the selector determine the scope of how the rules are applied.
